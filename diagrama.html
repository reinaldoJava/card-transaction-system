Fluxo (Hexagonal · Saga assíncrona · OpenTelemetry)
1Cliente → Login Function URL (credenciais)
2LoginUseCase valida no DynamoDB Users (BCrypt)
3Troca token opaco → JWT na Token-Exchange Lambda
4Cliente → Process Function URL (request + JWT)
5Orchestrator: idempotência + fraud score em cache
6Salva PENDING + injeta traceparent no SagaPayload
7StartExecution (Standard) e responde 202 + correlationId
8ValidateBusinessRules → AnalyzeFraud (Bedrock, tools internas)
9UpdateStatus: APPROVED ou compensa; grava idempotência
✗Erro em qualquer step → RollbackCompensation (Catch)
